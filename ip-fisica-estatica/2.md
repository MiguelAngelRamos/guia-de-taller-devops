Si quieres que tu servidor Ubuntu vuelva a pedir una IP automáticamente a tu router (vía DHCP) y deje de ser estática, es muy sencillo. Solo debemos indicarle a Netplan que active el `dhcp4`.

Aquí tienes los pasos para revertirlo:

### 1. Editar el archivo de configuración
Abre el archivo de Netplan:

```bash
sudo nano /etc/netplan/50-cloud-init.yaml
```

### 2. Dejarlo en modo DHCP
Borra todo el contenido actual y reemplázalo por esta configuración limpia. Es mucho más corta porque el router se encargará de darte la IP, la puerta de enlace y los DNS:

```yaml
network:
    version: 2
    ethernets:
        enp0s3:
            dhcp4: yes
```

> **Nota:** Asegúrate de mantener los espacios (sangría) tal cual están arriba, ya que los archivos `.yaml` son muy sensibles a eso.



---

### 3. Aplicar los cambios
Guarda los cambios con `Ctrl + O`, pulsa `Enter` y sal con `Ctrl + X`. Luego, aplica la configuración:

```bash
sudo netplan apply
```

---

### 4. Verificar que funcionó
Ahora revisa qué IP te asignó el router:

```bash
ip a
```

Deberías ver una dirección IP dentro del rango de tu red local (probablemente una distinta a la `.94` o `.61` que tenías antes).

### ⚠️ Un detalle importante para tu Jenkins
Como estás siguiendo el curso de **Jenkins**, ten en cuenta que si dejas la IP dinámica (DHCP), es posible que mañana tu router le asigne una IP diferente a esa máquina. Si eso ocurre:
1. El **Master** no podrá conectarse al **Slave** porque lo buscará en la IP vieja.
2. Tendrás que entrar a la interfaz de Jenkins y actualizar la IP en la configuración del Nodo.

Por eso, en servidores de infraestructura como los que estás montando, se prefiere la IP estática. Pero para pruebas rápidas o si tu router siempre asigna la misma IP por defecto, el DHCP está perfecto.
